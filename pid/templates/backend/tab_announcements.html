10/09/2017 - 1.0 Release!

We are proud to announce that PLAID version 1.0 is finally released!

Improvements (since 0.0.9-alpha)

  • Added emails to many steps in workflow/approval logic.
  • Added records awaiting approval to dashboard.
  • Change logs on all records have been majorly improved.
  • Improved UX of all modals. Mac Retinas in default resolution should be less of a problem now.

Bugfixes (since 0.0.9-alpha)

  • Record states are now selectable again in advanced searches.
  • Padawans can no longer remove supervisor from list of approvers.
  • Self-approved is now reset when a record is revised.
  • ECOs can no longer be added to closed Design records.
  • Software version field on Anomaly and ECO is now editable.

Misc (since 0.0.9-alpha)

  • Tons and tons of backend improvements.
  • Added links to PLAID space on Confluence as well as links to report bugs/request features in JIRA (links are in footer).
  • Moved database to PostgreSQL 9.6.5 (from 9.5).
  • Action icons that earlier would have opened an empty modal are now grayed out instead.

09/26/2017 - 0.0.9-alpha Release

Improvements

  • Added workflows and approvals to all records in PLAID.
  • Added padawan and supervisor capabilities to users.
  • Can now select specific revisions of designs when creating an anomaly or ECO.
  • Anomalies and ECOs now support multiple designs.
  • Anomalies can now be created against 'Other'.
  • Can now view build information and add new products/vendor products to existing builds.
  • Added approvals table to dashboard.

Bugfixes

  • Nothing major.

Misc

  • PLAID now runs over HTTP2 for a significant speed boost.
  • Revision log entries are now editable for open records.
  • Vendor product serial numbers are now editable.
  • A modal with a list of designs is now shown if a user creates multiple designs at once.
  • Added software version field to anomalies and as runs.

8/10/2017 - 0.0.8-alpha release

Improvements

  • Minor improvements to advanced search.
  • Minor improvements to tasks tab on dashboard.
  • Users are now sent an email when a task is assigned to them.
  • App now shows number of currently active users in footer.

Bugfixes

  • Fixed bug where users were unable to create new vendor products.
  • Fixed bug with permissions on procedures.

Misc

  • Celery and RabbitMQ implemented to handle backgrounding tasks.
  • Better and more informative CSRF token error handling.

7/11/2017 - 0.0.7-alpha release

Improvements

  • Improved search logic when adding components to a Part under a Design. Search will no longer return inseparable components not belonging to this specific design and this specific revision. Search will no longer return parts that already have added this specific part as an component (prevents circular reference). Search will no longer return parts that are under this specific design and revision (unless inseparable component, prevents incest).
  • Improved advanced search. UI/UX fixes, search indicator, etc.
  • Added internal PLAID settings page (only accessible by admins).
  • Buttons to sort Tasks on Dashboard now work.

Bugfixes

  • Forbidden revisions are now skipped app wide.
  • Better error checking with revise design form.
  • Fixed thumbnails not working on Task.

Misc

  • Added "Inspection" as a state to Product and Vendor Product.
  • Changed "Manufacturer" label to "Distributor".
  • Added "N/A" as default distributor.
  • Internal code cleanup.

6/29/2017 - 0.0.6-alpha release

Improvements

  • Added error feedback to form when user is creating an Anomaly or adding a Procedure to a Design.
  • Added datepicker to Tasks.
  • Advanced search in top nav is now available to all users.
  • Create build modals now get pre-populated with some information from design and part like project etc.
  • Made it more clear when a product is not at latest revision, or it's design is not released.
  • Added shortcut to create As-Run from the table of As-Runs on a Procedure.
  • Moved some fields around on Product and Vendor Product to make them more similar and information easier to find.
  • Improved global quick search with better ordering and now showing how many matches at bottom of results list.

Bugfixes

  • Fixed bug where user was unable to add parts to an As-Run during creation.
  • Fixed bug where opening a modal, closing it, then opening it again and submitting would result in incorrect page redirects.
  • Corrected as-built lists on products. Link to design is now to latest revision, description points to part, state points to installed product if any.
  • Serial number list on as-built list for product now includes serial numbers from other revisions of same design/part as well.
  • Fixed bug where thumbnail could not be changed on Tasks, Procedures and Specifications.
  • Fixed bug with documents, URLs and discrepancies not being able to update inline due to bugs with permissions logic.
  • Discrepancy table header now shows when user creates the first discrepancy.
  • References are now usable on Tasks.
  • Fixed bug where user was unable to add an Anomaly to a Vendor Part.
  • Fixed bug where Trumbowyg dropdown menus would not be on top of other elements.
  • Refactored part component ordering logic. User can no longer move components on top further up, or components on bottom further down.
  • Users can no longer open multiple instances of the same modal.
  • Revising a Design, Procedure or Specification now sets the new revision to 'In Work'.
  • Better parsing of 'weird' serial numbers for vendor products. All products should now be accessible.

Misc

  • Updated datepicker in app and improved it everywhere it's used.
  • Various style improvements.
  • Added initial input focus to most (all?) modals upon opening them.
  • Started adding loading indicators to modals. So far only added to "Add Extra Product Component" modal but will roll out to all in future.
